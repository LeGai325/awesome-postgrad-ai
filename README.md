# awesome-postgrad-ai

> 考研初试出分后全流程 AI 实战仓库：联系导师、陶瓷套磁、回复解读、复试跟进、进组确认、PPT 简历设计，以及学术写作 Prompt/Skills 整合。

---

## 📌 项目简介

这个仓库用于解决出分后的连续决策问题：如何定位、如何联系老师、如何解读回复、如何并行推进调剂与保底、如何确认进组、如何把经历整理成导师可快速判断的简历材料。同时，仓库也整合了学术写作 Prompt 与 Skills 内容，便于后续在邮件、研究计划、复试材料、论文写作中统一使用。

---

## 🧭 全流程地图（出分后 → 入学前）

1. 成绩定位与风险分层  2. 院校-导师池构建  3. 首轮联系导师  4. 陶瓷/套磁  5. 复试前沟通  6. 复试后跟进  7. 回复意图解读  8. 调剂并行  9. 确认导师与进组  10. 拟录取后协作维护  11. 入学前预研与能力补齐。

# Part I · Prompt 模板库

> 大家只需把中括号中的内容替换为个人信息即可。

## 1) 初次联系导师邮件 Prompt

你是一位熟悉国内研究生招生生态的学术沟通顾问，请根据以下单段信息生成两版邮件（正式版与简洁版），并给出三个标题备选。学生信息如下：[姓名]，[本科院校与专业]，[初试总分与排名情况]，[英语能力]，[科研经历]，[项目经历]，[技术栈]，[可提供附件]。导师信息如下：[导师姓名]，[学校学院]，[研究方向]，[近期论文或项目]，[公开招生偏好]。请确保邮件语气克制、真实、不夸大，不直接追问“是否有名额”，但要体现我与导师方向的具体匹配点（可点名论文或方法关键词），正文长度控制在 [220-320 字] 区间，结尾使用“若您方便，期待得到进一步建议”的礼貌表达。

示例：你是一位熟悉国内研究生招生生态的学术沟通顾问，请根据以下信息生成邮件。学生为张三，来自某 211 计算机专业，初试 372 分，参与过目标检测项目并负责训练与误差分析，熟悉 Python、PyTorch；导师方向为多模态检索，近两年发表跨模态鲁棒学习相关论文。请输出 3 个标题及两版邮件，强调我已阅读其工作并对噪声标签鲁棒性问题有持续兴趣。

## 2) 陶瓷（套磁）邮件 Prompt

你是研究生申请中的策略型沟通助手，请根据以下信息生成一封“轻触式陶瓷邮件”以及一封 7 天后的低打扰跟进邮件：我的背景是 [院校专业]、[初试分数]、[相关经历]；目标导师方向是 [方向]；我已读论文包括 [论文 A/B]；我可提供的可验证材料包括 [代码/报告/成绩单链接]。邮件必须避免催促名额、避免情绪化示弱、避免强行绑定关系，重点是表达长期研究兴趣和可执行能力，并在结尾请求“若您方便希望获得建议”。

示例：你是研究生申请中的策略型沟通助手，我来自双非计算机，初试 356，做过推荐系统课程项目并复现过对比学习 baseline，导师方向是 NLP 与推荐系统交叉，已阅读其用户行为建模论文。请写首封陶瓷邮件与 7 天后跟进邮件，跟进中只补充“新增复现结果”，不重复催问。

## 3) 复试前沟通 Prompt

你是复试沟通教练，请将以下单段进展信息改写成可直接发送的复试前同步邮件：我近期完成了 [专业课复盘内容]、[项目重现实验]、[英语准备情况]，当前短板是 [短板项]，计划在 [时间] 前补齐 [补齐动作]，希望向老师请教 [具体问题]。请输出一封 180-260 字的邮件，要求突出执行力与反馈闭环，不做空泛寒暄；另外请附上一条“若老师仅回复‘好好准备复试’时”的二次跟进文本。

示例：我完成了数据结构、操作系统高频题复盘，重跑了项目 ablation，英语口语在练习中，短板是论文阅读速度，计划 10 天内补齐。请教问题是“入学前是否建议重点补数学基础还是先做方向复现”。

## 4) 复试后跟进 Prompt

你是学术沟通文案顾问，请基于以下复试后信息生成两版感谢跟进邮件（标准版与低打扰版）：复试中我被问到 [问题点]，交流中导师提到 [建议点]，我目前意向强度为 [高/中/保留]，我已在复试后完成 [补充动作]。请控制语气礼貌克制，不逼问结果、不做情绪化表达，长度控制在 [180-260 字]，目标是表达尊重与持续意向，而非索要承诺。

示例：复试中导师关注我项目实验设计，建议我补充对比实验；我意向较高，已完成实验补充框架。请写两版跟进邮件，一版正式、一版简洁。

## 5) 导师回复意图解读 Prompt（核心）

你是熟悉导师筛选机制的研究生申请策略分析师，请对以下原始回复进行字面与策略双层解读：导师原文为 [粘贴回复]，当前时间节点为 [出分后/复试前/复试后/调剂期]，我与导师已互动 [次数] 次，我当前竞争位势为 [高/中/低]。请输出：该回复的表层含义、隐含意图概率判断、所属类型（明确欢迎/暧昧观察/拖延/筛选/委婉拒绝/明确拒绝/已有人选）、下一步动作建议、可直接发送的跟进文本、止损阈值（多长时间无反馈就转向）。

示例：导师回复“先准备复试，后续再联系”，时间节点为复试前，已互动 1 次，我位势中等。请判断更偏“观察型”还是“拖延型”，并给出 7 天后跟进文本及转向建议。

## 6) 调剂并行 Prompt

你是调剂策略助手，请根据以下信息生成“调剂联系邮件 + 3 天并行行动表 + 止损转向规则”：我的初试情况为 [分数/排名区间]，可接受院校层级与地域为 [范围]，研究偏好为 [方向]，当前时间窗口为 [天数]。请把输出写成可执行日程，明确每天投递数量、回复判断标准、优先级重排逻辑，以及无回复时的转向条件。

示例：分数 338，排名中下，可接受 211 与强双非，偏向数据挖掘方向，窗口 5 天。请给 3 天强执行计划。

## 7) 简历生成 Prompt

你是一位研究生申请简历优化顾问，请把以下原始素材改写为“可放入 PPT 的页级文案”：我的基本信息是 [姓名/联系方式]，教育背景是 [学校/专业/GPA/排名]，科研经历是 [课题/方法/贡献/结果]，项目经历是 [技术栈/指标/落地]，成果是 [论文/竞赛/奖项]，技能是 [语言与工具]，目标导师方向是 [方向]。请输出一页版（邮件附件）与三页版（教育/科研/项目）两套内容，每页不超过 6 条，每条尽量量化，按研究相关性排序，并附 60 秒口头自我介绍稿。

示例：某同学在 XX 课题中将 F1 从 0.81 提升至 0.87，并完成推理加速使时延下降 28%；请把这两项写成导师可快速识别的简历亮点语句。

## 8) 简历素材采集 Prompt

你是简历信息采集助手，请通过连续追问把我的素材整理为可上 PPT 的结构化内容：当前已知信息为 [粘贴现有经历]，请你先输出“教育/科研/项目/成果/技能”五栏草稿，再逐栏追问缺失项，重点追问可量化指标、个人真实贡献、可验证证据（代码/报告/证明），最后输出“可直接贴入 PPT 的最终文案”。

示例：已知我有 2 段课程项目、1 次竞赛获奖、无论文，请你通过追问把每段经历改写成“问题-方法-结果”三句结构。

# Part II · Agent Skills（整合版）

## 1) Contact-Agent

用于批量管理“导师池 + 联系策略 + 跟进节奏”。输入为学生画像、导师画像、历史往来文本，输出为首封邮件、跟进邮件、优先级重排建议与止损建议。示例：A 导师高优先 48h 内跟进，B 导师观察低频维护，C 导师止损转向。

## 2) Resume-Agent（PPT）

用于把原始经历抽取为“页级文案 + 数据化亮点 + 方向化重排”。输入为简历素材与目标方向，输出为一页版/三页版 PPT 内容草稿与口头介绍稿。示例：P1 个人定位，P2 研究结果，P3 项目落地与技能矩阵。

## 3) Reply-Interpreter-Agent

用于解析导师回复的真实意图与风险。输入为回复原文与时间节点，输出为分类标签、置信度、下一步动作、可发送跟进文本和止损阈值。示例：筛选型（0.72），24h 补材料，72h 无反馈转向。

## 4) Workflow-Agent

用于将全流程拆解成每日任务并生成周复盘。输入为目标院校与当前阶段，输出为日程、提醒、漏项与风险项。示例：本周完成率 78%，复试后跟进超时 2 位导师。

# Part III · 学术写作 Prompt + Skills

> 以下内容作为“读研沟通 + 学术写作”统一工具层。

## A. 写作 Prompt 场景清单

已整合场景：中转英、英转中、中转中、缩写、扩写、英文润色、中文润色、逻辑检查、去 AI 味、论文架构图、实验绘图推荐、图标题生成、表标题生成、实验分析、Reviewer 视角审视、模型选择。建议在邮件、研究计划、复试材料、开题文档中按场景调用。

### A1) 中转英（段落模板）

你是一位顶会审稿标准的学术写作助手，请将以下中文草稿翻译并润色为英文论文段落，保持 LaTeX 文本纯净、逻辑严谨、时态一致、术语准确，避免机械连接词和 AI 腔，特殊字符按 LaTeX 规则转义，数学公式保持原样；请输出英文正文与中文直译两部分，输入内容为 [粘贴中文草稿]。

### A2) 英转中（段落模板）

你是一位计算机领域学术翻译助手，请将以下英文 LaTeX 片段直译为中文，删除 \cite/\ref/\label 等索引命令，仅保留文本语义并把公式转成易读表达，不做润色扩写；输入为 [粘贴英文 LaTeX]。

### A3) 缩写（段落模板）

你是一位简洁性导向编辑，请在不丢失信息的前提下把以下英文 LaTeX 段落微幅缩减（约 5-15 词），保留参数与结论，输出缩减后正文、中文直译与修改日志；输入为 [粘贴英文段落]。

### A4) 扩写（段落模板）

你是一位学术写作扩展助手，请将以下要点扩展为逻辑完整的论文段落，补足动机、方法细节与结论衔接，避免空话与重复，保持术语一致；输入为 [粘贴要点]。

### A5) 逻辑检查（段落模板）

你是一位审稿视角的逻辑审查助手，请对以下段落进行论证链检查，指出前提缺失、因果跳跃、结论过度泛化、实验支撑不足之处，并给出可直接替换的改写句；输入为 [粘贴文本]。

### A6) 去 AI 味（段落模板）

你是一位自然学术表达优化助手，请将以下文本去除模板化连接词和重复句式，使语言更自然、紧凑、可读，同时不改变事实和结论；输入为 [粘贴文本]。

## B. Skills 配置与使用

可将常用写作任务封装为 skills，并在日常流程中采用“先模板生成、再人工审校、后批量执行”的策略。建议把高频模板放在 `skills/prompts/`，把成功案例放在 `skills/examples/`，把流程检查清单放在 `skills/checklists/`。

## C. 导师沟通策略补充

核心原则：联系导师不是一次性邮件任务，而是“匹配证明 + 节奏管理 + 风险对冲”的过程。应避免单点押注，至少维持主申与保底并行；每次跟进都应提供新增信息（进展、结果、材料）而非重复催促；对“礼貌但无动作”的回复设置止损时间。
