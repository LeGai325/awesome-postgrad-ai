---

## 📌 项目简介

这个仓库专门解决一个高频痛点：

**出分后你该怎么联系老师，如何判断老师回复真实意图，以及如何用 AI 快速产出“导师愿意看”的简历。**

本仓库采用双主线结构：

- **Part I：Prompt 模板库**（直接复制即可用）
- **Part II：Agent Skills**（可扩展为自动化工作流）

---

## 🎯 适用人群

- 初试出分后，准备联系目标导师的同学
- 分数不稳，准备“主申 + 保底 + 调剂”并行策略的同学
- 希望把简历从“信息堆砌”改为“研究潜力表达”的同学
- 想把流程标准化，减少焦虑和无效沟通的同学

---

## 🧭 全流程地图（出分后 → 入学前）

1. 成绩定位与风险分层
2. 院校-导师池构建
3. 首轮联系导师邮件
4. 陶瓷/套磁（名额不明阶段）
5. 复试前沟通
6. 复试后跟进
7. 导师回复意图解读
8. 调剂与保底并行
9. 确认导师与进组
10. 拟录取后协作维护
11. 入学前预研与能力补齐

# Part I · Prompt 模板库（含示例）

> 每个部分均包含：**用途、Prompt 模板、输入示例、输出示例**。

## 1) 初次联系导师邮件 Prompt

### 用途
用于出分后首次建立联系，目标是“进入可观察名单”，不是“立即要承诺”。

### Prompt 模板

```markdown
# Role
你是一位研究生申请沟通顾问，擅长将学生信息转化为导师可快速判断的邮件。

# Task
根据学生背景与导师信息生成首封联系邮件（正式版+简洁版）。

# Input
- 学生：姓名、本科院校、专业、初试分数、排名、科研/项目、技术栈
- 导师：姓名、研究方向、近期论文、招生偏好

# Constraints
- 不夸大，不虚构。
- 不追问“还有没有名额”。
- 必须体现具体方向匹配点。
- 正文 220~320 字。

# Output
1. 邮件标题（3个）
2. 正文（正式版）
3. 正文（简洁版）
```

### 输入示例

- 学生：某 211 计算机，本科 AI 方向，初试 372，参与 1 个目标检测项目。
- 导师：研究多模态检索，近两年发表 2 篇相关论文。

### 输出示例（节选）

- 标题：`张三-计算机专硕初试372-申请与您课题组沟通`
- 正文句式示例：`我阅读了您关于跨模态检索的近期工作，尤其关注其中对噪声标签鲁棒性的处理思路。`

## 2) 陶瓷（套磁）邮件 Prompt

### 用途
用于名额未明时建立“弱连接”，提升导师识别度。

### Prompt 模板

```markdown
# Role
你是陶瓷策略助手。

# Task
生成轻触式陶瓷邮件 + 7天后跟进短邮件。

# Input
- 学生背景
- 导师方向
- 已阅读论文与可展示成果

# Constraints
- 不催名额，不表忠心。
- 必须包含1个具体研究兴趣点。
- 语气礼貌克制。

# Output
- 首封邮件
- 跟进短邮件（<=120字）
```

### 输出示例（节选）

- 首封重点句：`若您方便，想请教该方向在研究生阶段优先建议补齐的能力。`
- 跟进重点句：`近期我补充实现了相关 baseline，若您愿意我可提交简短复现报告供您参考。`

## 3) 复试前沟通 Prompt

### 用途
展示执行力与准备状态，不做“空泛问候”。

### Prompt 模板

```markdown
# Role
你是复试沟通教练。

# Task
生成复试前进展同步邮件，突出准备进度和请教问题。

# Input
- 已完成内容
- 正在补足短板
- 希望请教的问题

# Output
- 邮件正文
- 导师简短回复后的二次跟进建议
```

### 输出示例（节选）

- `近两周我完成了数据结构与操作系统的高频题复盘，并对项目中的实验设置做了重现实验。`

---

## 4) 复试后跟进 Prompt

### 用途
在 24~72 小时内做礼貌跟进，表达意向但不施压。

### Prompt 模板

```markdown
# Role
你是学术沟通文案顾问。

# Task
生成复试后感谢邮件（标准版 + 低打扰版）。

# Input
- 复试简要复盘
- 与导师交流片段
- 入组意向强度

# Constraints
- 不逼问结果
- 180~260字
```

### 输出示例（节选）

- `感谢您在复试交流中对我项目思路的建议，我已按建议补充了对比实验设计。`

## 5) 导师回复意图解读 Prompt（核心）

### 用途
把“客气回复”转成可执行动作，避免误判窗口期。

### Prompt 模板

```markdown
# Role
你是研究生申请策略分析师。

# Task
分析导师回复的显性信息、隐含意图与行动建议。

# Input
- 导师回复原文
- 当前时间节点
- 历史互动次数
- 当前竞争位势

# Output
1. 表层意思
2. 隐含意图（概率）
3. 类型归类（7类）
4. 下一步动作
5. 跟进模板
6. 止损阈值
```

### 7 类回复类型（每类含示例）

1. **明确欢迎型**：`可以约时间详细聊下你做的项目。`
2. **暧昧观察型**：`先准备复试，后续再联系。`
3. **拖延型**：`现在不确定，到时候再说。`
4. **筛选型**：`把你的成绩单和项目报告发我看看。`
5. **委婉拒绝型**：`你背景不错，建议也联系下其他老师。`
6. **明确拒绝型**：`今年名额已满。`
7. **已有人选型**：`会优先考虑已有科研基础的同学。`

### 输出示例（节选）

- 结论：`当前更接近“筛选型”，建议24小时内提交结构化材料，不建议长篇解释。`

## 6) 调剂并行 Prompt

### 用途
用于高压时间窗口下的并行推进与止损。

### Prompt 模板

```markdown
# Role
你是调剂策略助手。

# Task
输出调剂邮件 + 3天行动清单 + 转向策略。

# Input
- 初试分数与排名区间
- 可接受地域/院校层级
- 研究偏好
- 时间窗口
```

### 输出示例（节选）

- Day1：完成目标池 A/B/C 分层并发送第一批邮件。
- Day2：根据回复进行优先级重排。
- Day3：对无回复对象执行止损转向。

## 7) 简历生成 Prompt（PPT 版本）

> 按你的要求，这里不再提供 LaTeX 模板，改为 **PPT/PPTX 简历方案**。

### 用途
把散乱经历整理成“导师 30 秒可读”的研究型简历页。

### Prompt 模板

```markdown
# Role
你是一位研究生申请简历优化顾问。

# Task
根据输入信息输出“PPT 简历内容草稿”（每页标题+要点+数据化表达）。

# Input
- 基本信息
- 教育背景
- 科研经历
- 项目经历
- 论文/竞赛/奖项
- 技术技能
- 目标导师方向

# Constraints
- 每页不超过6条要点。
- 每条要点尽量数据化。
- 按“研究相关性”排序。

# Output
1. 一页简历版本（适合邮件附件）
2. 三页简历版本（教育/科研/项目）
3. 口头介绍稿（60秒）
```

### 结构示例（可映射到你后续上传的 PPTX）

- Page 1：个人信息 + 方向标签 + 三条核心优势
- Page 2：科研经历（问题-方法-结果）
- Page 3：项目经历与技能矩阵

### 内容示例（节选）

- `在XX课题中负责数据清洗与模型训练，F1 从 0.81 提升至 0.87。`
- `实现推理加速方案，单次推理时延下降约 28%。`

# Role

# Part II · Agent Skills（含示例）

## 1) Contact-Agent（联系导师）

### 能力

- 自动生成首封邮件、跟进邮件
- 结合导师回复动态调整策略
- 管理并行导师池与沟通节奏

### 输入示例

- 学生：分数 372，NLP 方向，1 段科研 + 2 段工程项目
- 导师池：A/B/C 三位导师

### 输出示例

- A 导师：高优先，48h 内跟进
- B 导师：观察，低频维护
- C 导师：止损转向

---

## 2) Resume-Agent（PPT 简历）

### 能力

- 从原始经历抽取高价值要点
- 自动生成 PPT 页级文案草稿
- 按目标方向重排展示顺序

### 输出示例

- `P1标题：个人定位`
- `P2标题：研究经历与量化结果`
- `P3标题：项目落地与技术栈`

---

## 3) Reply-Interpreter-Agent（回复解读）

### 能力

- 对导师回复做“字面-意图-风险”三级拆解
- 生成“下一封该怎么写”的模板
- 给出止损阈值，避免无限等待

### 输出示例

- 分类：筛选型（置信度 0.72）
- 动作：24h 内补材料，72h 无反馈则转向

---

## 4) Workflow-Agent（全流程编排）

### 能力

- 将流程拆成每日任务
- 跟踪进度与漏项提醒
- 生成每周复盘报告

### 输出示例

- 本周任务完成率：78%
- 风险项：复试后跟进超时 2 位导师

## 🛠 快速使用（5 步）

1. 先填“素材采集 Prompt”拿到结构化个人信息。
2. 用“初次联系导师 Prompt”生成首批邮件。
3. 收到回复后立刻跑“意图解读 Prompt”。
4. 并行推进主申与调剂，不做单点押注。
5. 用“PPT 简历 Prompt”输出附件版本并持续迭代。

---

## ✅ 输出质量自检

- 邮件是否体现了“具体方向匹配”？
- 每条成果是否有可验证数据？
- 每次回复后是否有明确下一步动作？
- 是否设置了跟进截止时间与止损点？
