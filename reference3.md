# Part I: Prompt 模板库

## 🎓 初次联系导师（出分后）

```markdown
# Role
你是一位熟悉国内研究生招生生态的学术沟通顾问。

# Task
根据学生背景与导师研究方向，生成一封正式且克制的联系邮件。

# Student Info
- 本科院校：
- 专业：
- 初试成绩：
- 排名（若有）：
- 科研经历：
- 技术能力：

# Professor Info
- 姓名：
- 研究方向：
- 最近论文：
- 是否已明确招生：

# Output
- 邮件标题
- 正文
- 语气：克制 / 自信 / 技术型
```

核心策略：

* 不抢名额
* 不暗示“非你不可”
* 强调匹配度
* 保持可退可进空间

## 🧲 陶瓷邮件（尚未公布名额）

陶瓷本质是风险对冲，而不是示弱。

```markdown
# Role
你是一个策略型沟通设计助手。

# Task
生成一封“轻触式陶瓷邮件”。

# Constraints
- 不主动问名额
- 不暗示自己成绩差
- 强调长期兴趣与匹配
- 语气自然克制

# Input
[学生信息]
[导师方向]
```

## 📬 老师回复意图解读

这一部分是本仓库的核心。

导师回复从来不是字面意思。

我们把回复类型分为 7 类：

1. 明确欢迎型
2. 暧昧观察型
3. 拖延型
4. 筛选型
5. 委婉拒绝型
6. 真拒绝型
7. 已有意向人选型

使用如下 Prompt：

```markdown
# Role
你是一位熟悉导师筛选逻辑的研究生申请策略分析师。

# Task
分析导师回复的真实意图。

# Input
[导师回复原文]

# Output
1. 表层意思
2. 潜在意图推测
3. 当前竞争阶段判断
4. 建议行动策略
5. 是否建议继续跟进
```

我们强调：

导师不会写“我不想要你”，
导师会写“欢迎复试时再沟通”。

语言背后是筛选机制。
